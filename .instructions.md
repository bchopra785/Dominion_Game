# Group B Project Context

## Project Overview
This is a Java-based game engine and player implementation for a card/strategy game (appears to be based on Dominion or similar). Built with Maven and Spring Boot.

## Project Structure
- **Main code**: `src/main/java/edu/brandeis/cosi103a/groupb/`
  - `engine/` - Core game engine including card functions and board/player state
  - `network/` - Network communication layer (App.java, DecisionRequest/Response)
  - `player/` - Player implementations (BigMoneyPlayer, ConsolePlayer, etc.)
- **Tests**: `src/test/java/edu/brandeis/cosi103a/groupb/`
- **Build**: Maven with `pom.xml`

## Build & Runtime
- **Java Version**: 23
- **Build Tool**: Apache Maven 3.6.3+
- **Key Dependencies**:
  - Spring Boot 3.4.3 (web framework)
  - JUnit Jupiter 6.0.3 (testing)
  - Mockito 5.22.0 (mocking)
  - Jackson 2.15.2 (JSON serialization)
  - **ATG API 2.5.3** (custom game framework)
    - Javadoc: https://www.javadoc.io/doc/io.github.brandeis-cosi-103a/atg-api/2.5.3/index.html/
    - Provides core game engine abstractions
    - Already included in pom.xml; import directly in code

## Common Commands
- `mvn clean compile` - Build the project
- `mvn test` - Run all tests
- `mvn package` - Create JAR file
- `mvn -v` - Check Maven version

## Development Guidelines
- Target Java 23 compatibility (set in pom.xml properties)
- Maintain the existing package hierarchy
- Add unit tests for new features
- Use Mockito for testing with dependencies
- Follow existing naming conventions (e.g., CardFunctions use PascalCase)
- **DO NOT automatically run `WeightedPlayerOptimizationTest.java`** - This test is computationally heavy and should only be run manually when needed for optimization testing. It should not be included in standard test runs.
